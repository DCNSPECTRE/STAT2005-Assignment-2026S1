Perth’s freeway network has undergone major changes in recent years. One of the more controversial upgrades has been the Smart Freeways project; a modification which aims to improve the travel time of commuters from Perth’s outer suburbs heading towards the city. This system has been implemented on both the Kwinana Freeway Northbound and the Mitchell Freeway Southbound, controlling the flow of traffic predominantly through digital speed-limit gantries and on-ramp metering. On the Mitchell South, this system saves 7-minutes when travelling from Hester Avenue to Vincent Street in the morning peak (Main Roads, 2025). However, residents closer to the city are generally more impacted by ramp-metering, and, have noticed their morning commute more than doubling in length (ABC, 2025).

In order to weigh the validitity of these claims, this study will analyse the time spent queuing the enter the freeway and compare this with previous research of the time saved on the mainline. We will perform parameter optimisation to adjust and maximise release timing while maintaining stability of the system.

A queuing system with \(c\) lanes is stable (i.e. has a controllable load) if its utilisation rate \(\rho = \frac{\lambda}{c \times \mu}\) is less than \(1\), where \(\lambda\) and \(\mu\) refer to the mean arrival and departure rates respectively (Righini, 2020). In terms of ramp-metering, this utilisation rate is typically maintained in a range that is safely below capacity as a means of risk aversion, or avoiding a total breakdown of the queue system (Levinson, 2021).

TALK ABOUT MEMORYLESS PROPERTY AS A FALLBACK/ASSUMPTION

In order to construct a queuing model, we can use this dataset to simulate the incoming volume of traffic during the morning peak. Typically, this model would be formed on the Poisson distribution because of its memoryless property and ease of implementation. However, a Poisson distribution assumes an equal mean and variance, so it’s important that verify this assumption. To visualise, let’s filter the above for weekdays, the peak hours, and without public holidays (eg 06/03/23 – Labour Day), yielding the following:

The volume of traffic entering the freeway varies heavily over the course of the morning. It would be unreasonable to fit a single model to this level of variation, so instead consider a piecewise approach. Models that account for the minimum, maximum and mean volume each morning will be constructed, correlating to the times 6am, 7am and approximately 8am.

We’ll focus on the 6:00-6:15am interval for now:

              Mean Variance
6am (min) 113.2273  161.803

This is an overdispersed dataset where the variance of volume is considerably higher than its mean. This is true throughout the entire dataset, meaning a Poisson distribution would be unsuitable as we do not satisfy the assumption of equidispersion. In order to provide a better fit, we can consider a Negative Binomial where both a mean and a size parameter (for dispersion) can be selected. (NEED TO TALK ABOUT HOW MEMORYLESS IS STILL INVALID AND ALSO ASSUMPTION OF INDEPENDENCE REMAINS) We can calculate this size parameter \(r\) as follows:

\[r = \frac{\mu^2}{\sigma^2 - \mu}\]

To see the improvement in fit, let’s simulate a Poisson and a Negative Binomial with these parameters and compare with the original dataset.

The Negative Binomial is a much better fit to the original dataset. However, a new concern arises: for the purpose of simulation, we wish to simulate vehicle-counts on a per-minute basis. This will help to increase the granularity of our queuing time results. One possible solution (Method 1) is to simply divide the volume data by 15mins and fit a new model. If we consider this case, our mean will scale by a factor of \(\frac{1}{15}\), and the variance by a factor of \(\frac{1}{15^2}\):

                 Mean  Variance      Size
Vehicles/min 7.548485 0.7191246 -8.343333

Since this is now underdispersed, a Negative Binomial will no longer fit (hence the negative size). It remains unbiased, but erases so much of the original variance that it is completely irrelevant. An alternative solution (Method 2) that maintains our original variation is to use the divisible property of the Negative Binomial. We first form the model, then scale its parameters to maintain a dispersion that remains in proportion to the new, lower mean.

This occurs because the Negative Binomial is formed as a Compound Poisson (Siegrist, 2022), so if \(X=X_1 + X_2 +...+X_n \sim NBinom(k, p)\) then we can model each \(X_i\) as \(NBinom(\frac{k}{n}, p)\) with the same probability \(p\). Extrapolating this \((_{successes}, \space _{probability})\) form to the \((_{mean}, \space _{size})\) form yields the parameters \((\frac{\mu}{n}, \space \frac{r}{n})\).

T <- 1000 # no. of timesteps (each is a 15min interval)
c <- 2 # no. of lanes

mu <- 5.8 * 15 # mean no. vehicles released PER LANE in one timestep 
lambda <- smu8 # mean no. vehicles arriving TOTAL in one timestep
dispersion <- ssize8 # dispersion parameter for arrival rate

q1 <- c()
q2 <- c()

wtimes <- list()
q1c <- numeric(T)
q2c <- numeric(T)

avg_wait_at_i <- list()

for (i in 1:T) {
  # arrivals
  arrivals <- rnbinom(1, size = dispersion, mu = lambda)
  
  q1a <- floor(arrivals / 2)
  q2a <- ceiling(arrivals /2)
  
  q1 <- c(q1, rep(i, q1a))
  q2 <- c(q2, rep(i, q2a))
  
  # release parameters
  release <- mu
  timestep.wtimes <- c()
  
  # finding how many cars should be released
  q1r <- min(length(q1), release)
  q2r <- min(length(q2), release)
  
  # releasing cars and tracking how long they waited
  if (q1r > 0) {
    timestep.wtimes <- c(timestep.wtimes, i - q1[1:q1r])
    q1 <- q1[-(1:q1r)]
  }
  if (q2r > 0) {
    timestep.wtimes <- c(timestep.wtimes, i - q2[1:q2r])
    q2 <- q2[-(1:q2r)]
  }
  
  # record data
  wtimes[[i]] <- timestep.wtimes
  q1c[i] <- length(q1)
  q2c[i] <- length(q2)
  
  avg_wait_at_i[i] <- if(length(timestep.wtimes) > 0) mean(timestep.wtimes)
  
}
